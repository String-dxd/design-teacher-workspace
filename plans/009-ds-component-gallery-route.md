## Why this matters

There is no design-system / component-reference page in the app today (a previous
Flow-DS `/ds` playground was deliberately deleted in merged plan 006). But the
building blocks of a Shadcn/Base-UI gallery already exist and are **imported
nowhere**:

- `src/components/component-example.tsx` (496 lines) — a showcase of the app's UI
  kit (AlertDialog, Badge, Button, Card, Combobox, DropdownMenu, Field, Input,
  Select, Textarea) with a light/dark switcher. Exports `function ComponentExample()`
  (line 96), which self-wraps in `<ExampleWrapper>`.
- `src/components/example.tsx` — `Example` / `ExampleWrapper` gallery-layout
  primitives (titled cells with dashed borders).

Wiring these to a `/ds` route is a near-free win: it gives the team a living
reference for the design system and makes token regressions (like the
disabled-button bug fixed in plan 007) visible instead of latent. Verified clean:
`component-example.tsx` imports only `@/components/ui/*`, `@/components/example`,
`react`, and `lucide-react` — no `@flow/*`, nothing deleted by plan 006.

## Current state

### Route convention (from `src/routes/insight-buddy.tsx` and peers)

Routes are file-based (TanStack Router). A minimal route:

```tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/<path>')({
  component: <ComponentFn>,
})
```

The route component renders inside the app shell (`__root.tsx` — sidebar +
breadcrumbs). `src/routeTree.gen.ts` is auto-generated by the TanStack Router
Vite plugin on `dev`/`build` and is committed.

### The component to mount

`src/components/component-example.tsx`:

```tsx
'use client'
...
export function ComponentExample() {   // line 96
  return (
    <ExampleWrapper>
      ... <Example title="...">...</Example> (many) ...
    </ExampleWrapper>
  )
}
```

It is self-contained — no required props.

## Steps

### Step 0: Baseline

`npx tsc --noEmit 2>&1 | grep -c "error TS"` (≈113), `bun run build` (exit 0).
Confirm `grep -n "export function ComponentExample" src/components/component-example.tsx`
→ matches (line ~96). If the export name differs, adjust the import in step 1 to match.

### Step 1: Create `src/routes/ds.tsx`

```tsx
import { createFileRoute } from '@tanstack/react-router'
import { ComponentExample } from '@/components/component-example'

export const Route = createFileRoute('/ds')({
  component: ComponentExample,
})
```

### Step 2: Regenerate the route tree and verify

Run `bun run build` (regenerates `src/routeTree.gen.ts` to include `/ds`).

**Verify**:
- `grep -c "'/ds'" src/routeTree.gen.ts` → ≥ 1 (route registered).
- `npx tsc --noEmit 2>&1 | grep -c "error TS"` → ≤ 113.
- `bun run dev`, then `curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:3000/ds` → `200`.
- If `agent-browser` is available: open `/ds`, confirm the gallery renders (a grid
  of titled example cells with Buttons, Inputs, a Card form, dialogs). A first
  visit may show a "Welcome to Teacher Workspace" modal — dismiss it first.
